# Yaad Surface Analysis

## 1. Test Patterns

### Test File
- **File**: `integration_test.go` (1 file, ~1200 lines)
- **Package**: `integration_test` (black-box, external package tests)
- **Test count**: 38 tests

### Test Helper Pattern
```go
func setup(t *testing.T) (*engine.Engine, func()) {
    t.Helper()
    dir := t.TempDir()
    store, err := storage.NewStore(filepath.Join(dir, "test.db"))
    // ...
    eng := engine.New(store)
    return eng, func() { store.Close(); os.RemoveAll(dir) }
}
```

Key characteristics:
- **Fresh DB per test**: `t.TempDir()` creates isolated temp directories
- **Cleanup via closure**: returned `func()` calls `store.Close()` + `os.RemoveAll(dir)`
- **Called with `defer cleanup()`** in every test
- **No test fixtures** — all test data is created inline within each test function
- **No shared DB state** — tests are fully independent
- **No subtests** — all tests are top-level `TestXxx` functions
- **No table-driven tests** (except `TestPhase6IntentClassifier` and `TestUtilsShortID` which use `[]struct` cases)

## 4. CORS Status

**Minimal**: Only 1 CORS header set in the entire codebase:

- `internal/server/sse.go:46` — `w.Header().Set("Access-Control-Allow-Origin", "*")` on the SSE endpoint only

No CORS middleware exists for REST API routes. No `Access-Control-Allow-Methods`, `Access-Control-Allow-Headers`, or preflight (`OPTIONS`) handling anywhere. The SSE endpoint is the only one accessible cross-origin.

---

## 5. Signal Handling Status

**No signal handling exists anywhere in the codebase.**

No `SIGTERM`, `SIGINT`, `signal.Notify`, `os.Signal`, or `signal.Stop` calls found. The server (`ListenAndServe`) starts and blocks indefinitely with no graceful shutdown path. The TUI handles `ctrl+c` internally via Bubbletea's `tea.Quit` command, but the main REST server and gRPC server have no OS signal handlers.

## 6. Config Loading Status

**`config.Load()` is never called in `main.go` or anywhere else in production code.**

- `internal/config/config.go` has both `Load()` and `Default()` implemented
- `Load()` reads from `~/.yaad/config.toml` (global) and `.yaad/config.toml` (project-specific)
- **Only** `config.Default()` is referenced, and only in `integration_test.go:1137` (`TestConfigDefaults`)
- `openEngine()` in `cmd/yaad/helpers.go` directly creates `storage.NewStore(dbPath())` without loading any config
- The CLI `serve` command uses a hardcoded `:3456` address flag, not config-driven
- `NewRESTServer` takes an address string, not a config struct

## 7. Code Quality Hotspots

### Bubble Sort in `sortByScore` (`internal/engine/memory.go:175-185`)
```go
func sortByScore(nodes []*storage.Node) {
    for i := 0; i < len(nodes); i++ {
        for j := i + 1; j < len(nodes); j++ {
            if sj > si {
                nodes[i], nodes[j] = nodes[j], nodes[i]
            }
        }
    }
}
```
This is an O(n²) bubble/selection sort. Used in:
- `internal/engine/memory.go:175` — `sortByScore` (bubble sort)
- `internal/engine/search.go:114-115` — temporal recency path, insertion sort

### AllEmbeddings Memory Concern (`internal/storage/vectors.go:49-50`)
```go
func (s *Store) AllEmbeddings() (map[string][]float32, error)
```
This loads ALL embeddings into memory at once. For large graphs with many vector embeddings, this could consume significant memory. Called from:
- `internal/engine/search.go:126` — `hybridSearch.vectorSearch()` loads all embeddings for every vector search query
- `integration_test.go:198` — test usage

### RRF Fusion Complexity (`internal/engine/search.go`)
The hybrid search function is complex (4-path retrieval: BM25, vector, intent-aware graph, temporal) with ~100 lines. The RRF constant is hardcoded (`rrfK = 60`).

### Magic Numbers
- `internal/engine/search.go:9` — `rrfK = 60` (RRF constant, undocumented)
- `internal/engine/search.go:94` — temporal sort uses a raw insertion sort for `queryIntent == IntentWhen`
- `internal/engine/proactive.go:35,48,78` — various hardcoded limits (5, 10)
- `internal/compact/compact.go:17` — `maxTokens = 50000` default
- `internal/engine/entities.go` — entity extraction with hardcoded types

### No Error Handling for Embedding Operations
- `hybridSearch.Search()` ignores errors from `h.store.SearchNodes()` silently
- `vectorSearch` silently returns nil on `AllEmbeddings` error
- `ProactiveContext.Predict()` ignores errors from `ListNodes`, `GetNeighbors`, etc.

- `cmd/yaad/main.go:47` — hardcoded `:3456` address in root command
- `cmd/yaad/main.go:255` — hardcoded `:3456` in serve command flag default
- `internal/server/mcp.go:45` — hardcoded embedding provider model string
